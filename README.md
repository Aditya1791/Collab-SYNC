## рҹҢҢ Why Collab-SYNC?

Most project tools feel dead вҖ” you submit a change and pray it saved. **Collab-SYNC** is the opposite. It's a living, real-time workspace where every action вҖ” moving a card, editing a task, joining a board вҖ” is instantly reflected for every connected user, no refresh required.

```
  Before Collab-SYNC          With Collab-SYNC
  в”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җ          в”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җ
  вқҢ "Did my change save?"    вң… Changes sync in <100ms
  вқҢ Who is working on what?  вң… Live presence avatars
  вқҢ Forgot my password       вң… OTP reset via phone
  вқҢ Privacy = all or nothing вң… Per-field privacy controls
  вқҢ Flat, boring profiles    вң… Cover photo + avatar + bio
  вқҢ One workspace for all    вң… Multi-workspace hierarchy
```

## вңЁ Features

<details open>
<summary><b>рҹ”җ Authentication & Security</b></summary>
<br/>

- **Sign Up / Login** вҖ” Email + password with server-side JWT issuance
- **Two-Factor Authentication (2FA)** вҖ” Optional OTP verification via:
  - `рҹ“§` Email OTP вҖ” code sent to your registered email
  - `рҹ“ұ` SMS OTP вҖ” code sent to your linked phone number
- **OTP Password Reset** вҖ” Enter phone number вҶ’ receive OTP вҶ’ reset password. No old password needed
- **JWT Sessions** вҖ” Stateless Bearer tokens; enforced on every API call
- **Protected Routes** вҖ” Unauthenticated users are fully gated from all app pages

</details>

<details>
<summary><b>рҹ‘Ө Rich User Profiles</b></summary>
<br/>

- **Username** вҖ” Unique display name with cooldown-based edit protection
- **Email** вҖ” Primary identifier and OTP delivery channel
- **Phone Number** вҖ” Optional; enables SMS OTP and phone-based password recovery
- **Avatar / Profile Picture** вҖ” Upload custom JPG/PNG or choose a color preset
- **Cover Photo** вҖ” LinkedIn-style banner; uploadable image or solid color preset
- **Profile Color** вҖ” Accent fallback shown when no avatar image exists
- **Initials Avatar** вҖ” Auto-generated from username as a graceful fallback

</details>

<details>
<summary><b>рҹ”’ Granular Privacy Controls</b></summary>
<br/>

Every sensitive field has **three independently configurable visibility levels**:

| Level | Symbol | Who Sees It |
|-------|--------|-------------|
| Everyone | рҹҢҚ | Any user on the platform |
| Friends | рҹ‘Ҙ | Only shared workspace members |
| None | рҹҡ« | Nobody вҖ” shows as **NIL** |

Privacy is set per-field, independently for:
- рҹ–јпёҸ Profile Picture / Avatar
- рҹ“§ Email Address  
- рҹ“ұ Phone Number

</details>

<details>
<summary><b>рҹҸў Workspaces & Role-Based Access</b></summary>
<br/>

- **Multi-Workspace Support** вҖ” Create unlimited workspaces for different teams or clients
- **Workspace Ownership** вҖ” Owners have full admin privileges
- **Role-Based Access Control** вҖ” Three tiers enforced server-side (not just hidden UI):

| Role | Create Boards | Edit Tasks | Manage Members |
|------|:---:|:---:|:---:|
| рҹ‘‘ Admin | вң… | вң… | вң… |
| рҹ§‘вҖҚрҹ’ј Member | вң… | вң… | вқҢ |
| рҹ‘ҒпёҸ Viewer | вқҢ | вқҢ | вқҢ |

- **Email Invitations** вҖ” Invite users by email with role assignment at point of invite

</details>

<details>
<summary><b>рҹ“Ӣ Kanban Boards & Tasks</b></summary>
<br/>

**Boards:**
- Create unlimited boards per workspace (e.g. Design, Dev, Marketing)
- Fully customizable columns вҖ” add, rename, reorder
- Column order persists per board and restores on reload

**Task Cards include:**
| Field | Description |
|-------|-------------|
| рҹ“қ Title | Concise task name |
| рҹ“„ Description | Rich notes and context |
| рҹҸ·пёҸ Priority | `Low` / `Medium` / `High` / `Urgent` вҖ” color coded |
| рҹ“… Due Date | Deadline with overdue flag |
| рҹ‘Ө Assignees | One or multiple workspace members |
| вң… Checklist | Sub-items with individual toggles + live progress bar |

</details>

<details>
<summary><b>рҹ“Ў Real-Time Collaboration Engine</b></summary>
<br/>

Powered by **Socket.io** WebSockets вҖ” all events are instant, bidirectional, and room-scoped:

```
  User A drags a card  в”Җв”Җв–¶  Socket event emitted
                              в”Ӯ
                              в–ј
                        Server broadcasts to
                        all users in that board room
                              в”Ӯ
                              в–ј
  User B, C, D see  в—Җв”Җв”Җ  Card moves on their screen
  the card move            in real-time
```

**Events handled:**
- `task_order_updated` вҖ” drag-and-drop card moves
- `task_added` вҖ” new task created by any user
- `task_changed` вҖ” task edits (title, desc, priority, checklist)
- `task_removed` вҖ” task deletions
- `board_refreshed` вҖ” column additions or deletions
- `board_presence` вҖ” live user join/leave tracking
- `join_board` / `leave_board` вҖ” room management

</details>

<details>
<summary><b>рҹ”ҙ Live Presence System</b></summary>
<br/>

- **Active User Avatars** вҖ” Row of live user icons in the Navbar for the current board
- **Click-to-Preview Popover** вҖ” Click any avatar to open a centered profile card showing:
  - Cover photo background
  - Profile picture or initials
  - Username, Email, Phone (all subject to privacy rules)
- **Privacy-Aware** вҖ” Fields restricted to None/Friends show **NIL** to unauthorized viewers
- **Real-Time Updates** вҖ” Presence list updates as users join and leave

</details>

<details>
<summary><b>рҹҺЁ UI & Design System</b></summary>
<br/>

- **Glassmorphism** вҖ” Frosted glass panels, layered transparency, blurred backdrops
- **Dark / Light Mode** вҖ” Detected from system preference; persisted in `localStorage`
- **Micro-Animations** вҖ” Hover states, modal entrances, interactive feedback on every element
- **Responsive Layout** вҖ” Collapsible sidebar; Navbar adapts gracefully to screen width
- **Color-Coded Priorities** вҖ” Instantly recognizable visual hierarchy on task cards
- **Accessible Fallbacks** вҖ” Initials avatars and color placeholders prevent broken image states

</details>

## рҹҡҖ Quick Start

**Prerequisites:** Node.js v18+ В· npm v9+

```bash
# Clone
git clone https://github.com/Aditya1791/Collab-SYNC.git
cd Collab-SYNC

# Install
npm install

# Configure
cp .env.example .env
# Fill in your values in .env

# Run
npm run dev
```

Open **[http://localhost:3000](http://localhost:3000)** вҖ” frontend and backend both start together.
